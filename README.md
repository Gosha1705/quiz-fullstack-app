## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## 🧪 QA & Edge Case Scenarios (Testing)

As per the requirements, here are 5 custom edge-case scenarios that have been tested to ensure the application's resilience:

1. **Edge Case: Direct Access to Results Page**
   - *Action:* User manually types `/results` in the URL bar without taking the quiz.
   - *Expected:* App shouldn't crash trying to render undefined data.
   - *Actual Result:* The app checks `localStorage`. Finding it empty, it automatically redirects the user back to the `/quiz` page. (Pass ✅)

2. **Edge Case: Backend API is Down (Network Error)**
   - *Action:* User completes the quiz, but the NestJS backend is offline.
   - *Expected:* User should see a friendly error, not a blank screen or a crashed app.
   - *Actual Result:* The frontend catches the `TypeError: Failed to fetch`, safely stops the loading state, and displays a standard browser alert notifying the user of the connection issue. (Pass ✅)

3. **Edge Case: Empty Message Submission (Validation)**
   - *Action:* User tries to send a message to an advisor without filling in their name or email.
   - *Expected:* The form should prevent submission and prompt for required fields.
   - *Actual Result:* Native HTML5 `required` attributes block the submission. The submit function is never called, preventing empty DB records. (Pass ✅)

4. **Edge Case: Missing Advisor ID in Dynamic Route**
   - *Action:* User navigates to a non-existent advisor profile (e.g., `/advisor/99999`).
   - *Expected:* Graceful fallback if the fetch fails.
   - *Actual Result:* The component remains in the "Загрузка профиля..." (Loading) state or handles the empty data safely without throwing a fatal React error. (Pass ✅)

5. **Edge Case: Mobile Responsiveness on Profile Cards**
   - *Action:* Viewing the matched advisors on a device with a screen width < 640px.
   - *Expected:* The 3-column grid should collapse into a single-column layout so text remains readable.
   - *Actual Result:* Tailwind's `grid-cols-1 md:grid-cols-3` utility perfectly snaps the cards into a vertical stack on mobile devices. (Pass ✅)
